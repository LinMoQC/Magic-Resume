# Agent 工具授权前端速览

> 本文只记录前端公开边界：什么时候展示授权、UI 如何表达、用户决定如何回到当前会话。后端暂停/续跑、工具执行、鉴权实现以后端私有文档为准，不在前端公开仓复制。

## 目标

- 敏感动作必须先让用户看见“AI 想做什么”和“为什么需要”。
- 用户可以允许、拒绝，必要时可编辑授权范围。
- 授权卡是会话内的一条消息，不打断上下文，也不自动读取用户数据。
- 授权结果只表达用户决定；前端不暴露后端运行时、内部线程或工具实现。

## 前端状态

| 状态 | UI 行为 |
|---|---|
| `pending` | 展示授权卡，主操作为允许，次操作为拒绝。 |
| `approved` | 锁定卡片，显示已允许，并等待后续流式结果。 |
| `denied` | 锁定卡片，显示已拒绝，AI 继续用非敏感上下文回答。 |
| `expired` | 展示已过期或会话已结束，允许用户重新发起请求。 |

## 展示信息

授权卡只展示用户能理解的内容：

- 动作名称：例如“读取当前简历”。
- 原因：一句话解释这次为什么需要。
- 范围：尽量用产品语言描述数据范围。
- 后果：允许后会继续当前任务；拒绝后不会读取该数据。

不要展示内部工具名、后端路由、线程 id、鉴权细节或运行时实现。

## 前端接线

- 授权事件进入统一的 AI 会话事件消费层。
- `AiChatShell` 负责把事件落成会话消息和后续状态变更。
- `ChatThread` 只负责渲染授权卡和触发用户决定。
- service 层只暴露语义化方法，例如 `approveTool(...)`，组件不拼接路由。

## 验收标准

- [ ] AI 读取用户简历前，界面必须先出现授权卡。
- [ ] 拒绝授权后，前端不展示“正在读取”或“已读取”的状态。
- [ ] 授权卡文案不包含内部后端路由、端口、服务名或实现细节。
- [ ] 授权中的请求可随会话关闭被中止，避免继续消耗用户配置的模型额度。
