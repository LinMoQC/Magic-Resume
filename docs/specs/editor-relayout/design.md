# 设计简报:编辑器结构性重构 ——「舞台优先」(Stage-First Editor)

> 产出自第二轮 `/impeccable shape` 发现访谈。本文件是**设计计划**,不含代码——它让后续实现精准。
> 上游依据:`.impeccable.md`(Design Context)+ `docs/specs/ai-lab-living-canvas/design.md`(已定稿就地标注语言)。
> 交接给:`/impeccable craft` / `/impeccable` / 任意实现流程。
>
> **本版取代上一版「增量整理」方案。** 上一版只在五栏骨架里给浮动坞分组、把文档动作上移;经第二轮访谈,作者选择**结构性重构 + 就地编辑为主**,因此骨架本身被重排。增量版里仍然成立的部分(文档动作上顶部、AI 改右侧、状态语义、文案、动效)被吸收进本版。

---

## 1. 功能概述

把简历编辑器从**五条竖带**(图标轨 → 300px 表单 → 中间画布 → 一列圆按钮 → 模板栏)重排为**「画布是舞台」的就地编辑工作台**。核心病根有四,作者全部确认:

- **一墙同款圆按钮** —— 旗舰 AI 与边角缩放长得一样,找不到主动作(`Tools.tsx`:9 个 `w-8 h-8` 圆点)。
- **空间分配倒挂** —— 低频的「选择模板」占永久大列,高频的内容编辑挤在 300px。
- **画布太小** —— 真正的产品(简历)缩到 60%、四面被 UI 夹着,是屏幕上最小最暗的东西。
- **编辑割裂** —— 主编辑器只读 + 左栏填表;而能**就地编辑**的 living-canvas 被关在 `/ai-lab` 另一条路由里,像两套产品。

面向为目标岗位打磨简历的中文求职者:他们需要专注、长时段的创作台,且**一眼找到「问 AI」和「导出」**,改简历像改文档一样**点着就改**。

## 2. 首要用户动作

**「在简历上就地改 / 唤起 AI 协作」** 是画布上视觉权重最高的单一动作链。其次是「导出成稿」。其余(换模板、缩放、历史、反馈)是支撑工具,可达但不抢戏。

判断标准:新用户进入编辑器后,**3 秒内能(a)直接点简历上的文字开始改,(b)指出「问 AI 在哪」「怎么导出」**。

## 3. 设计方向

延续 `.impeccable.md`:深色工作台(`#0A0A0A`)、sky 蓝点缀(`#38bdf8`)、少 border、动效轻快贴元素(仅 transform/opacity,不弹跳)、「实验室」隐喻。
本次重构落地 **品牌原则 1 与 4**:

- **画布是舞台,对话是旁白** —— 简历放大成主角,内容就地改;AI/模板退为按需浮现的旁白与抽屉,绝不常驻抢舞台。
- **就地,而非另开** —— 编辑与 AI 改动都标注在简历本身上(白底红删 `#dc2626` / 绿增 `#15803d`),不再把「能编辑的画布」藏进另一条路由。

反面参照:把编辑做成横穿屏幕的远程填表、把 AI 做成花哨通用 FAB、模板/对话喧宾夺主占满舞台、卡片堆砌、冷表单。

## 4. 布局策略

核心思路:**五条竖带 → 两条常驻(薄大纲轨 + 画布)+ 顶部条 + 按需右抽屉**。空间向简历倾斜,所有非编辑功能改为「按需浮现」而非「常驻 chrome」。

```
┌──────────────────────────────────────────────────────────────────┐
│ ⌂  Kairo's Resume   ·  ◐ 已同步               导出 PDF  分享  ⋯   │ ← 顶部条:文档级动作右锚
├──┬───────────────────────────────────────────────────┬───────────┤
│§ │                                                   │           │
│§ │                ┌─────────────────────┐            │  右抽屉   │
│§ │                │                     │            │ (默认收起 │
│§ │                │      简历 = 舞台      │           │  画布最宽)│
│§ │                │   就地编辑 · 红删绿增  │   ← 放大   │           │
│§ │                │                     │            │ 模板 ⇆ AI │
│§ │                └─────────────────────┘            │  互斥共用 │
│＋│                                      [ ✦ AI 助手 ] │           │ ← sky 胶囊(主)
│  │                                      [ ＋ − ⊙ ]    │           │ ← 缩放分段(次)
└──┴───────────────────────────────────────────────────┴───────────┘
 ↑ 薄大纲轨:模块跳转 + 当前高亮 + 「＋ 添加模块」
```

**四个落点:**

1. **左侧薄大纲轨(常驻,极窄 ~56–64px)** —— 复用现有左轨(`SidebarNav`),精简为「结构/导航」一族:模块图标跳转、滚动时当前模块高亮、底部 `＋ 添加模块`。**300px 表单栏(`ResumeContent`)整条删除**——它的编辑职责搬到画布,`查看 JSON`/`保存` 移到顶部溢出菜单。

2. **中间画布 = 舞台(弹性,最大化)** —— 把 `/ai-lab` 里的 living-canvas 就地编辑(`LivingCanvas` / `ActionPopover`)**搬进主编辑器**,替换只读的 `ResumePreview`。点文字即改;模块间悬停出 `＋`;拖拽重排模块。AI 改动与人工改动统一就地标注(红删/绿增),逐条接受/拒绝。

3. **顶部条 HeaderTab = 文档级动作** —— `导出 PDF`(显眼主动作)、`分享` 移到标题/同步状态旁(类比 Figma/Docs 右上);`版本历史`、`反馈`、`查看 JSON` 收进 `⋯` 溢出菜单。**同步状态**重做成有语义的指示(同步中/已同步/失败可重试),替换当前裸文字。

4. **右侧 = 按需抽屉(模板 ⇆ AI 互斥,默认收起)** —— 模板面板**不再永久占列**:需要时从右滑出缩略图网格,选定即可收起 → 修复「空间倒挂」。AI 胶囊点开 → 同一处右抽屉滑出对话**旁白**栏(消息 + 输入 + 起点快捷)。**同时只开一个**,切换用快速滑动不闪烁。

**视觉节奏:** 极窄轨(深色,近乎隐形)→ 大画布(最大留白,简历居中放大)→ 收起的抽屉边。打破「等距等大圆点」的单调,用**单一 sky 胶囊**作为画布上唯一的主色锚点制造层级。

## 5. 关键状态

**画布(可编辑舞台)**
- `read/idle`:干净简历,无 chrome。
- `hover-block`:文字块浮现可编辑暗示(光标 + 极淡高亮),不喧哗。
- `editing`:就地文本编辑(Tiptap),内联极简工具条。
- `hover-gap`:模块间浮现 `＋ 添加模块`;模块出现拖拽手柄可重排。
- `empty`(首次/空简历):**教界面的空状态**——脚手架引导补第一个模块,而非空白页发憷。
- `ai-proposal`:AI 改动就地标注(红删/绿增),每条 `接受 / 拒绝 / 看理由`(复用 `ActionPopover`)。
- `ai-thinking`:实验室运行中(`AiThinkingOverlay`,动效轻),可中断。

**左侧薄大纲轨**
- `default`:模块图标列,当前模块随滚动高亮。
- `hover`:浮出模块名 tooltip。
- `＋`:`添加模块`菜单。

**右抽屉(互斥)**
- `closed`(默认):画布最宽。
- `templates`:缩略图网格滑入;选中后可自动收起。
- `ai-empty`(首次):起点快捷选项(优化某段 / 按 JD 改写 / 翻译 / 精简到一页),永远可自由输入/跳过。
- `ai-error`:说发生什么 + 可重试,动词开头、不复述。

**顶部文档动作**
- 同步:`同步中…` / `已同步` / `同步失败,点击重试`。
- 导出:`导出 PDF` → `导出中…` → `已导出`。
- 分享:`生成分享链接` → `链接已复制`。

**画布角落**
- AI 胶囊:`idle`(克制呼吸)/ `hover`(微提亮+微缩放)/ `active`(抽屉在场,胶囊变收起把手)。
- 缩放分段:常驻次级,放大/缩小/重置。

## 6. 交互模型

- **就地编辑是主回路**:进入即画布优先 → 点文字直接改(乐观更新,即时反映)→ 模块间 `＋` 增、拖拽排、大纲轨跳转。无需横穿屏幕去左栏。
- **AI 协作回路**:点 sky 胶囊 → 右抽屉滑入旁白栏(~200–300ms ease-out)→ 输入/选起点 → AI **就地**在简历上提案(红删/绿增)→ 逐条接受/拒绝 → 收起回全画布。改动全程在简历上可见,对话退为旁白(原则 1、4)。
- **右侧互斥**:开 AI 时模板抽屉收起,反之亦然;快速滑动切换。
- **渐进披露**:画布默认只露 AI 胶囊 + 缩放;文档动作在顶部;低频项(历史/反馈/JSON)在溢出菜单。
- **乐观更新**:编辑即时反映,同步后台跑,状态在顶部如实显示(含失败可重试)。
- **悬停披露次级动作**:模块工具条、抽屉次级操作 hover 才出现,默认干净。

## 7. 文案需求(语气:简短、动词开头、不复述已见)

- 画布添加:`＋ 添加模块`;模块空态:`补一段{模块名}`(而非「暂无」)。
- AI 胶囊:`AI 助手`(或 `问 AI`)。运行中:`实验室运行中…`。
- AI 空状态起点:`按目标岗位优化这段`、`让这段更有量化结果`、`翻译成英文`、`精简到一页`。
- 同步:`已同步` / `同步中…` / `同步失败,点击重试`。
- 导出:`导出 PDF` → `导出中…` → `已导出`。
- 分享:`生成分享链接` → `链接已复制`。
- AI 错误:`这段没改成,换个说法再试`(说发生什么 + 下一步,不说「我无法…」)。

## 9. 待解问题(实现时定夺)

1. **本地/自托管模式的 AI 可见性**【本轮明确暂缓】:当前 `isCloudMode` 门控;本轮只按 cloud 设计理想布局,本地模式是否给 BYOK 入口或降级态(「舞台中心缺席」如何处理)往后定。
2. **living-canvas 搬迁边界**:`/ai-lab` 现为独立路由,搬进主画布后,AI lab 路由是保留为「全屏专注模式」还是完全并入主编辑器右抽屉?
3. **大纲轨 ⇆ 画布重排的同步**:轨内拖拽是否镜像画布拖拽;两处排序的单一真源。
4. **保存模型**:表单栏删除后,`保存` 走全自动保存还是顶部保留显式按钮;`查看 JSON` 归溢出菜单还是开发者入口。
5. **模板抽屉行为**:选中模板后自动收起 vs 常驻到手动关。
6. **左轨现状核对**:实现前确认 `SidebarNav` 当前承载(app chrome:logo/语言/头像 vs 模块导航),区分「全局壳」与「大纲轨」两层职责。
7. **Dashboard 视觉统一**:本轮聚焦编辑页;列表页与之统一的视觉(色/间距/卡片质感/动效)另开一轮,信息架构不动。
