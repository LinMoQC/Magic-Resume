# AI 实验室 · 引导创建

> 本稿只定义前端 UX、状态、组件和文案边界。涉及 agent 运行、敏感数据读取、工具执行和后端事件的实现，留在后端私有文档中维护。

## 设计目标

- 从“空白简历”进入一个可对话、可预览、可评审的创建体验。
- 不把流程做成固定表单步骤，而是让 AI 用动态计划表达当前进展。
- AI 读取用户已有简历前，必须先展示授权卡。
- 右侧舞台始终有反馈：骨架、段落生成、改动高亮或错误兜底。

## 非目标

- 不在前端仓记录后端工具、事件、暂停/续跑或授权实现。
- 不把内部接口、端口、服务名写进 UX 文档。
- 不让 UI 组件直接承担数据读取或服务编排。

## 主要界面

| 区域 | 作用 |
|---|---|
| 左侧对话 | 提问、动态计划、授权卡、快捷回复 chips。 |
| 右侧舞台 | 简历草稿、骨架占位、局部生成高亮、后续可评审改动。 |
| 底部输入 | 用户继续补充目标、经历、语气或约束。 |

## 关键状态

| 状态 | 前端表现 |
|---|---|
| 入场 | 展示温暖开场和可选 chips。 |
| 等待首响 | 线程尾部出现“思考中”占位，避免空窗。 |
| 生成中 | 左侧流式文字，右侧对应区域 shimmer。 |
| 需要授权 | 会话中插入授权卡，解释动作与范围。 |
| 生成草稿 | 右侧逐步呈现简历结构和已生成内容。 |
| 出错/超时 | 保留已输入内容，展示可重试的兜底文案。 |

## 授权 UX

- 开场种子不包含用户简历正文。
- 授权卡只描述“想读取什么”和“为什么需要”。
- 用户拒绝后，AI 应继续基于当前对话工作。
- 授权组件复用 [Agent 工具授权前端速览](../../reference/agent-tool-approval-contract.md)。

## 组件落点

- `AiChatShell`：会话编排、等待态、授权状态、结果落地。
- `ChatThread`：消息、授权卡、动态计划和快捷回复渲染。
- `ArtifactCanvas` / `LivingCanvas`：右侧草稿和评审舞台。
- service 层：屏蔽请求、流式事件和响应归一化细节。

## 文案原则

- 少说系统状态，多说用户下一步能做什么。
- 不使用内部工具名、服务名或接口名。
- 授权文案必须清楚、短、可拒绝。
- 生成失败时给用户可继续的选择，而不是暴露错误堆栈。

## 验收标准

- [ ] 发送后立刻出现等待态。
- [ ] 读取已有简历前必须出现授权卡。
- [ ] 右侧舞台在空白、生成中、完成、失败时都有明确状态。
- [ ] UI 文案和注释不包含后端内部实现细节。
