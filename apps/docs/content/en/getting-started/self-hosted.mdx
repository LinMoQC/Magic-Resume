# Self-Hosted

In self-hosted mode the web app runs entirely in the browser. Resume data is persisted in IndexedDB under the database name `MagicResumeDB`. There is **no auth, no Core API, no Python agent, and no product analytics delivery** — AI features that need a backend are simply disabled until you configure one.

This is the recommended mode for hacking on the codebase.

## Prerequisites

- Node.js 20+
- pnpm `10.28.1` (the version in `packageManager`). Other versions work but lockfile drift is on you.

<Steps>

### Clone the repository

```bash
git clone https://github.com/LinMoQC/Magic-Resume.git
cd Magic-Resume
```

### Install dependencies

```bash
pnpm install
```

Run this from the **repo root** — pnpm needs to see `pnpm-workspace.yaml` to resolve `workspace:*` deps.

### Configure environment variables

```bash
cp apps/web/.env.example apps/web/.env.local
```

The minimum for self-hosted is:

```env filename="apps/web/.env.local"
NEXT_PUBLIC_APP_MODE=self-hosted
```

If `NEXT_PUBLIC_APP_MODE` is unset and `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY` is also unset, the app auto-detects `self-hosted`. Setting it explicitly is just clearer.

### Start the dev server

```bash
pnpm run dev
```

Turbo will spin up every workspace that has a `dev` script. The web app listens on [http://localhost:3000](http://localhost:3000); the docs site (this one) is on [http://localhost:3001](http://localhost:3001).

To run only the web app:

```bash
pnpm --filter @magic-resume/web dev
```

</Steps>

## What works without a backend

| Feature | Self-hosted |
|---|---|
| Resume editor, all 12 templates, export PDF | ✅ |
| Local IndexedDB storage | ✅ |
| Version history (local only) | ✅ |
| Bring-your-own-key AI in Settings | ✅ |
| Clerk auth, cloud sync, sharing | ❌ |
| Interview / JD analyze / translate (Python agent) | ❌ unless you run `apps/agent` or point `BACKEND_URL` somewhere |

<Callout type="info">
For the bring-your-own-key flow, open **Settings → AI** in the web app and paste an OpenAI / Anthropic / Gemini key. The key never leaves the browser.
</Callout>

## Environment reference

| Variable | Required | Purpose |
|---|---|---|
| `NEXT_PUBLIC_APP_MODE` | No | `self-hosted` or `cloud`. Auto-detected from the presence of a Clerk key. |
| `BACKEND_URL` | No | Python agent base URL. Only needed for AI Lab features. |

See [Cloud Mode](/getting-started/cloud) for the full cloud-mode env list.
