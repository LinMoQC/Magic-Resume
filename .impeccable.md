## Design Context

> 来源:`design/ai-lab-living-canvas.md`(已定稿 UX 语言)+ `.plan/AI能力使用手册` + 与作者的方向确认。代码只说明"建了什么",本节说明"给谁、该有什么感觉"。

### Users
Magic Resume 的用户是正在为某个目标岗位打磨 / 搭建简历的求职者(以中文市场为主)。「引导创建(create)」入口专门服务"从零开始"或想另起一份的人——既有经验少、需要被引导的应届 / 转行者,也有想快速出稿的有经验用户。进入这个流程时用户常面对"空白页发憷",需要明确的下一步和正向反馈。其余能力(优化 / 分析 / 翻译 / 面试)面向已有简历、准备投递的人。

### Brand Personality
呆萌外表·靠谱内核——一只憨态可掬的像素小蓝宠(方眼睛、天线星、会蹦跳),说话温暖、带一点俏皮;但给的建议永远具体、诚实、专业。轻松的是语气,不是内容:俏皮不进建议本体,开场问候可以尽情展现小宠性格。
语气:温暖自然,不强求简短(不注水、不复述用户已见);UI 微文案仍保持简短、动词开头。
情绪目标:让用户有掌控感与被理解感——像有个可靠的小伙伴陪着改,而不是被填表、也不是被 AI 牵着走。

### Aesthetic Direction
深色工作台(`#0A0A0A`)、sky 蓝点缀(`#38bdf8`)、少 border / 减少割裂;简历修订沿用白底红删(`#dc2626`)/ 绿增(`#15803d`)。动效轻、快、贴着元素,仅用 transform / opacity,不弹跳、不 elastic。「实验室」隐喻(FlaskConical)。深色源自:这是专注、长时段的创作工作台。
Anti-references:花哨空洞的动效 / 卡片堆砌、冷冰冰的表单、AI 自顾自发挥的不可控感、冗长问卷式交互。

**浅色主题(可切换,深色仍默认)**:概念=「日光下的实验台」——**暖纸桌 + 冷 sky 墨**。neutrals 向暖纸微染(OKLCH hue≈85,刻意不向品牌冷色染),sky 拆两支:`--ink-sky`(≥sky-700,承担文字/边框,AA 达标)与 `--glow-sky`(仅装饰)。深色靠发光造层次,浅色靠**清晰 hairline 边界 + 落地暖阴影**(glow → grounded light);简历画布浅深两态恒定近白(所见即所得)。AI 活画布元素(Polaris 球/edge-flow/thinking 遮罩/aura)在浅色下重设计为原生版:球落纸、描边清晰、纸磨砂、光晕大幅降透明。完整规范见 `docs/specs/light-theme/`(brief + design)。

### Design Principles
1. 画布是舞台,对话是旁白——工作发生在简历上,对话退为入口 / 旁白 / 历史。
2. 能力按需浮现,不做常驻 chrome。
3. AI 是合作者不是推土机——改动先提案、可拒绝、有理由;流程可跳过、可掌控。
4. 就地,而非另开——改动标在简历本身上,不另开 Diff / 弹窗。
5. 脚手架不是栅栏——给起点 / 快捷选项降低发懵成本,但永远允许自由表达与跳过。
